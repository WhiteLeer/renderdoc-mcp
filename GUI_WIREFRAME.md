# renderdoc-workbench GUI 线框说明

## 1. 目的

这份文档只定义第一期主窗口的结构、状态和交互，不讨论具体技术栈实现细节。

当前产品边界：

- 工具控制目标会话
- RenderDoc 自己完成快捷键抓帧
- 工具负责 RDC 刷新、分析和导出

因此主界面必须服务于这条最短路径，而不是把所有潜在能力平铺到首页。

## 2. 首页目标

用户第一次打开窗口后，应当在一个视图里完成下面的事情：

1. 选定 MUMU 或目标目录
2. 选定 package
3. 启动并注入 RenderDoc
4. 刷新 RDC 列表
5. 选中一个 RDC
6. 分析并导出

## 3. 主窗口结构

建议采用三栏加底部日志区。

```text
+----------------------------------------------------------------------------------+
| renderdoc-workbench                                              [会话状态]      |
+---------------------------+----------------------------------+-------------------+
| 目标与输入                | 当前 RDC / 操作                 | 当前状态          |
|                           |                                  |                   |
| MUMU 目录                 | [打开 RDC]                       | RenderDoc 状态    |
| [路径输入] [浏览]         | [分析 RDC]                       | 目标进程状态      |
|                           | [导出报告]                       | 当前 package      |
| package 列表              | [导出资源]                       | 当前 RDC          |
| [搜索]                    |                                  | 输出目录          |
| [下拉列表/列表]           | 分析摘要预览                     | 最近任务结果      |
|                           |                                  |                   |
| [启动并注入]              | 事件/资源/Pass 摘要              |                   |
|                           |                                  |                   |
| RDC 根目录                |                                  |                   |
| [路径输入] [浏览] [刷新]  |                                  |                   |
| 本地 RDC 列表             |                                  |                   |
| [列表]                    |                                  |                   |
+----------------------------------------------------------------------------------+
| 日志 / 错误 / 后台任务进度                                                      |
+----------------------------------------------------------------------------------+
```

## 4. 区块职责

### 左栏：目标与输入

这是会话准备区，只回答两个问题：

- 我要控制哪个目标
- 我要分析哪个 RDC 目录

包含控件：

- `MUMU 目录`
- `浏览`
- `package 搜索`
- `package 列表`
- `启动并注入`
- `RDC 根目录`
- `刷新`
- `本地 RDC 列表`

交互原则：

- 目标输入和 RDC 输入都放左栏，不要拆到多个页面
- `启动并注入` 属于目标区，不属于分析区
- `刷新` 属于 RDC 区，不属于工具栏

### 中栏：当前 RDC / 操作

这是主工作区，只处理当前选中的 RDC。

包含控件：

- `打开 RDC`
- `分析 RDC`
- `导出报告`
- `导出资源`
- `分析摘要预览`

交互原则：

- 这里不出现 package 选择
- 这里不出现注入控制
- 所有按钮都针对“当前选中的 RDC”

### 右栏：当前状态

这是上下文面板，用于减少用户的状态不确定感。

展示内容：

- RenderDoc 状态
- 当前目标进程
- 当前 package
- 当前 RDC 文件名与路径
- 输出目录
- 最近一次分析结果

交互原则：

- 以只读信息为主
- 允许复制路径
- 不承载复杂操作按钮

### 底栏：日志与后台任务

用于显示：

- 启动与注入日志
- RDC 刷新结果
- 分析进度
- 导出结果
- 错误信息

交互原则：

- 保留滚动日志
- 错误需要明确颜色和级别
- 产物路径应可点击

## 5. 第一期开启/禁用规则

### 初始状态

- 未选择 MUMU 目录时：
  - `启动并注入` 禁用
- 未选择 RDC 根目录时：
  - `刷新` 禁用
- 未选中 RDC 时：
  - `打开 RDC`、`分析 RDC`、`导出报告`、`导出资源` 禁用

### 会话已启动但未发现 RDC

- `启动并注入` 可再次执行，但需要提示可能重建会话
- `刷新` 启用
- 分析与导出仍禁用

### 选中 RDC 后

- `打开 RDC` 启用
- `分析 RDC` 启用
- `导出报告` 和 `导出资源` 可根据分析状态决定是否直接启用

## 6. 按钮文案

建议使用短动作名，不要用说明句。

推荐文案：

- `浏览`
- `启动并注入`
- `刷新`
- `打开 RDC`
- `分析 RDC`
- `导出报告`
- `导出资源`

不建议使用这类文案：

- `下拉列表，显示本次抓取的所有RDC`
- `注入按键，将指定package运行并注入renderdoc`

## 7. 第一期开窗即见内容

用户打开窗口后第一屏必须直接看到：

- 当前工具名
- MUMU 目录输入
- package 列表
- `启动并注入`
- RDC 根目录
- `刷新`
- RDC 列表
- `分析 RDC`

不要把这些入口藏到 tab 或二级弹窗里。

## 8. 以后再做的内容

这些内容不应进入第一期首页：

- 多页面导航
- 图表化渲染统计
- 自动诊断向导
- 批量分析看板
- 插件式侧边面板
- 大量美术化装饰

第一期首页要像工具控制台，而不是展示页。
